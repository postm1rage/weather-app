# Weather App

Приложение для просмотра погоды с возможностью сохранения списка городов, сортировки, фильтрации по температуре и просмотра статистики. Семестровая работа по дисциплине «Разработка клиентских веб-приложений».

## API

Используется [OpenWeatherMap](https://openweathermap.org/api) — эндпоинт `/data/2.5/weather`.  
Бесплатный тариф позволяет до 60 запросов в минуту.

## Запуск

1. Откройте проект через локальный сервер (ES-модули не работают при открытии `index.html` как файла):
   - в VS Code с расширением Live Server, или
   - через консоль: `npx http-server` (порт по умолчанию 8080).
2. Перейдите по адресу `http://127.0.0.1:8080` (или другому, выданному сервером).

Для удобства проверки API-ключ уже встроен в `js/data/weatherApi.js` (строка `const API_KEY = '...'`).  
При необходимости вы можете заменить его на собственный, полученный на [openweathermap.org](https://openweathermap.org/).

## Деплой на GitHub Pages

1. Убедитесь, что репозиторий публичный (или приватный с доступом для проверяющего).
2. В настройках репозитория выберите **Pages** → ветка `main`, папка `/ (root)` → **Save**.
3. Через минуту сайт будет доступен по адресу `https://ваш_логин.github.io/имя_репозитория/`.

Приложение готово к работе сразу после деплоя — ключ уже включён в код.

## Архитектура

Код разделён на два слоя:

- **Слой данных** (`js/data/`) — отвечает за общение с API, не зависит от DOM.
- **Слой отображения** (`js/ui/`) — компоненты интерфейса, каждый отвечает за свой блок на странице.

Связующим звеном выступает `App` (`js/app.js`) — он создаёт компоненты и соединяет их через колбэки, не занимаясь вёрсткой.

### Основные модули

| Файл | Класс | Ответственность |
|---|---|---|
| `js/data/weatherApi.js` | `WeatherService` | Выполняет запрос к OpenWeatherMap, обрабатывает ошибки, приводит данные к удобному формату. |
| `js/ui/welcome.js` | `WelcomeScreen` | Экран приветствия с полем ввода имени и кнопкой «Далее». |
| `js/ui/header.js` | `Header` | Шапка приложения: приветствие по имени, кнопка смены темы и кнопка «Сменить имя». |
| `js/ui/searchPanel.js` | `SearchPanel` | Панель поиска города, кнопка «Узнать погоду» и кнопка «Сохранить город». Управляет сообщениями об ошибках и состоянием загрузки. |
| `js/ui/weatherCard.js` | `WeatherCard` | Отображает карточку с текущей погодой (температура, влажность, ветер, описание, иконка). |
| `js/ui/controlsPanel.js` | `ControlsPanel` | Панель управления: выпадающий список сортировки и поля для фильтрации по температуре. |
| `js/ui/cityList.js` | `CityList` | Хранит массив сохранённых городов, реализует фильтрацию, сортировку и отрисовку таблицы. При изменениях оповещает подписчиков через `onUpdate`. |
| `js/ui/stats.js` | `Stats` | Блок статистики: количество городов, средняя/минимальная/максимальная температура. |
| `js/app.js` | `App` | Координатор: создаёт все компоненты, связывает их колбэками, переключает экраны. |

## Технологии

- **ES-модули** (import/export) — модульная структура без сборщиков.
- **Классы** — каждый компонент оформлен как класс с инкапсуляцией своего DOM.
- **async/await** — асинхронные запросы к API.
- **fetch** — выполнение HTTP-запросов.
- **Обработка ошибок** — try/catch в слое данных и в UI.
- **DOM-события** — click, change, keyup.
- **CSS-переменные/классы** — переключение между светлой и тёмной темами.

## Пользовательский функционал

1. Экран приветствия: ввод имени, кнопка «Далее».
2. Основной экран: приветствие по имени, поиск погоды по названию города.
3. Карточка погоды: отображение текущей температуры, влажности, ветра, иконки погоды.
4. Сохранение города в список (повторные города не добавляются).
5. Список городов в виде таблицы: можно удалять отдельные записи.
6. Сортировка по названию, температуре, влажности, ветру (через select).
7. Фильтрация по диапазону температуры (два числовых поля).
8. Статистика в реальном времени: количество городов, средняя/минимальная/максимальная температура.
9. Переключение темы (светлая / тёмная) — кнопка в шапке.

## Примечание

Для работы API может потребоваться включённый VPN, если доступ к openweathermap.org ограничен провайдером. В противном случае приложение выдаст ошибку сети.
